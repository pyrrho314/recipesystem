fringe_correct¶

gempy.science.gmosScience.fringe_correct(adInputs, fringes, fl_statscale=False, scale=0.0, statsec='', outNames=None, suffix=None, log=None, logName='gemini.log', logLevel=1, noLogFile=False)[source]¶

This primitive will scale and subtract the fringe frame from the inputs. It utilizes the Python re-written version of cl script girmfringe now called rmImgFringe in gmosTools to do the work.

NOTE: String representing the name of the log file to write all log messages to can be defined, or a default of ‘gemini.log’ will be used. If the file all ready exists in the directory you are working in, then this file will have the log messages during this function added to the end of it.

FOR FUTURE This function has many GMOS dependencies that would be great to work out so that this could be made a more general function (say at the Gemini level) .

Parameters:
  • adInputs (Astrodata objects, either a single or a list of objects) – Astrodata input(s) to be fringe corrected
  • fringes (AstroData objects in a list, or a single instance. Note: If there is multiple inputs and one fringe provided, then the same fringe will be applied to all inputs; else the fringes list must match the length of the inputs.) – Astrodata input fringe(s)
  • fl_statscale (Boolean) – Scale by statistics rather than exposure time
  • statsec (string of format ‘[EXTNAME,EXTVER][x1:x2,y1:y2]’ default: If CCDSUM = ‘1 1’ :[SCI,2][100:1900,100:4500]’ If CCDSUM = ‘2 2’ : [SCI,2][100:950,100:2250]’) – image section used to determine the scale factor if fl_statsec=True
  • scale (real) – Override auto-scaling if not 0.0
  • outNames (String, either a single or a list of strings of same length as adInputs.) – filenames of output(s)
  • suffix (string) – string to postpend on the end of the input filenames (or outNames if not None) for the output filenames.
  • log (A gemLog object from astrodata/adutils/gemLog.py . It is an upgraded version of the Python logger for use with all new scripts in gemini_python/ . Note: the logName, logLevel and noLogFile will be automatically determined from the logger object passed in to ‘log’ with the ScienceFunctionManager.startUp() function.) – logger object to send log messges to
  • logName (String, None causes default to be used.) – Name of the log file, default is ‘gemini.log’
  • logLevel (integer from 0-6, 0=nothing to screen, 6=everything to screen. OR the message level as a string (ie. ‘critical’, ‘status’, ‘fullinfo’...)) – verbosity setting for the log messages to screen, default is ‘critical’ messages only. Note: independent of logLevel setting, all messages always go to the logfile if it is not turned off.
  • noLogFile (Python boolean (True/False)) – A boolean to make it so no log file is created
