markHistory¶

class gempy.instruments.geminiTools.ScienceFunctionManager(adInputs=None, outNames=None, suffix=None, log=None, logName=None, logLevel=1, noLogFile=False, funcName=None, combinedInputs=False)[source]¶

A manager class to hold functions for performing input checks, naming, log instantiation... code that is repeated throughout all the ‘user level functions’ in the gempy libraries (currently those functions in science/geminiScience.py and gmosScience.py).

markHistory(adOutputs=None, historyMarkKey=None)[source]¶

The function to use near the end of a python user level function to add a historyMark timestamp to all the outputs indicating when and what function was just performed on them, then logging the new historyMarkKey PHU key and updated ‘GEM-TLM’ key values due to historyMark.

Note: The GEM-TLM key will be updated, or added if not in the PHU yet, automatically everytime wrapUp is called.

Parameters:
  • adOutputs (Either a single or multiple astrodata instances in a list.) – List of astrodata instance(s) to perform historyMark on.
  • historyMarkKey (Under 8 character, all caps, string. If None, then only ‘GEM-TLM’ is added/updated.) – The PHU header key to write the current UT time
