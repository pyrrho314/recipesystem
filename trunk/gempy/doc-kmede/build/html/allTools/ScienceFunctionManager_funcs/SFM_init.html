__init__¶

class gempy.geminiTools.ScienceFunctionManager(adInputs=None, outNames=None, suffix=None, funcName=None, combinedInputs=False)[source]¶

A manager class to hold functions for performing input checks, naming, log instantiation... code that is repeated throughout all the ‘user level functions’ in the gempy libraries (currently those functions in science/geminiScience.py and gmosScience.py).

__init__(adInputs=None, outNames=None, suffix=None, funcName=None, combinedInputs=False)[source]¶

This will load up the global variables to use throughout the manager functions and instantiate the logger object for use in here and back in the ‘user level function’ that is utilizing this manager.

Either a ‘main’ type logger object, if it exists, or a null logger will be returned from startUp() along with the checked adInputs and outNames.

Parameters:
  • adInputs (Astrodata objects, either a single or a list of objects. At least one object MUST be passed in for adInputs.) – Astrodata inputs to have DQ extensions added to
  • outNames (String, either a single or a list of strings of same length as adInputs.) – filenames of output(s)
  • suffix (string) – String to add on the end of the input filenames (or outNames if not None) for the output filenames.
  • funcName (String) – Name of the Python function using this manager.
  • combinedInputs (Python boolean (True/False)) – A flag to indicated that the input images of adInputs will be combined to form one single image output. The use of this parameter is optional and is overridden by providing outNames.
