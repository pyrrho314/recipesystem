add_var¶

gempy.science.geminiScience.add_var(adInputs, outNames=None, suffix=None)[source]¶

This function uses numpy to calculate the variance of each SCI frame in the input files and appends it as a VAR frame using AstroData.

The calculation will follow the formula: variance = (read noise/gain)2 + max(data,0.0)/gain

NOTE: Either a ‘main’ type logger object, if it exists, or a null logger (ie, no log file, no messages to screen) will be retrieved/created in the ScienceFunctionManager and used within this function.

Parameters:
  • adInputs (Astrodata objects, either a single or a list of objects) – Astrodata inputs to have DQ extensions added to
  • outNames (String, either a single or a list of strings of same length as adInputs.) – filenames of output(s)
  • suffix (string) – string to add on the end of the input filenames (or outNames if not None) for the output filenames.
